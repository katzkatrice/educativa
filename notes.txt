Saya sudah siapkan repo ini untuk deploy ke Vercel. Saya menambahkan vercel.json, server.php, E:/websiteku/             educativa.vercelignore, dan script build vercel di composer.json. server.php akan melayani aset statis dari public/     lalu meneruskan request lain ke Laravel, dan vercel.json mengarahkan semua route ke runtime PHP Vercel.                                                                                                                                         Verifikasi lokal:                                                                                                       php -l server.php lolos.                                                                                                npm run build juga berhasil, jadi aset Vite bisa dibangun untuk production.                                                                                                                                                                     Agar deploy benar-benar jalan di Vercel, Anda masih perlu set Environment Variables di dashboard Vercel:                APP_KEY                                                                                                                 APP_URL                                                                                                                 DB_CONNECTION, DB_HOST, DB_PORT, DB_DATABASE, DB_USERNAME, DB_PASSWORD                                                  Jika fitur auth email dipakai: MAIL_MAILER, MAIL_HOST, MAIL_PORT, MAIL_USERNAME, MAIL_PASSWORD, MAIL_FROM_ADDRESS                                                                                                                               Catatan penting:                                                                                                        Vercel cocok untuk Laravel ringan seperti repo ini, tapi storage lokal, session file, queue worker, dan database        SQLite lokal tidak cocok untuk production serverless. Saya sudah set SESSION_DRIVER=cookie dan CACHE_STORE=array di     vercel.json supaya auth dasar lebih aman di Vercel, tetapi database tetap harus pakai layanan eksternal.                                                                                                                                        Langkah deploy:                                                                                                                                                                                                                                 1. Push repo ini ke GitHub.                                                                                             2. Import repo ke Vercel.                                                                                               3. Isi semua env di atas.                                                                                               4. Deploy.                                                                                                              5. Jalankan migration ke database production secara terpisah, misalnya dari lokal dengan env production.                                                                                                                                        Sumber yang saya pakai:                                                                                                 https://vercel.com/docs/project-configuration/vercel-json                                                               https://github.com/iagobruno/deploy-laravel-on-vercel                                                                                                                                                                                           Kalau Anda mau, langkah berikutnya saya bisa bantu:                                                                                                                                                                                             1. menyiapkan .env production yang cocok untuk Vercel                                                                   2. menambahkan panduan deploy ke README.md                                                                              3. menyesuaikan config database/mail agar aman untuk provider tertentu seperti Neon, PlanetScale, Supabase, atau           Railway  




 Saya sudah paksa konfigurasi project ini agar tidak dibaca sebagai proyek Vite statis:                                                                                                                                                          vercel.json                                                                                                                                                                                                                                     - "framework": null                                                                                                     - "buildCommand": "npm run build"                                                                                       - "outputDirectory": "public/build"                                                                                                                                                                                                             Ini penting karena error dist biasanya muncul saat Project Settings di Vercel masih menganggap framework-nya Vite.                                                                                                                              Yang perlu Anda cek di dashboard Vercel juga:                                                                                                                                                                                                   1. Framework Preset ubah ke Other                                                                                       2. Build Command kosongkan atau set npm run build                                                                       3. Output Directory kosongkan atau set public/build                                                                     4. Root Directory pastikan memang root repo ini                                                                                                                                                                                                 Kalau setting dashboard masih Vite, kadang itu tetap menang terhadap config repo sampai Anda ubah manual lalu           redeploy.
